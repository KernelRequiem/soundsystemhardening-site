# Contribuer

Merci de votre intérêt pour ce projet. Toute contribution utile au mouvement est bienvenue, qu'elle soit juridique, technique ou rédactionnelle.

## Contribuer au wiki

Corriger ou enrichir une page ne demande aucune compétence en développement :

1. Ouvrez le dossier `src/content/wiki/` sur GitHub.
2. Modifiez un fichier `.md` existant, ou créez-en un nouveau via l'interface GitHub.
3. Proposez la modification (Pull Request) sur la branche `main`.

Les fichiers Markdown sont la source de vérité : aucune connaissance de l'outillage n'est nécessaire pour écrire ou corriger une page.

## Contribuer au code

1. Forkez le dépôt.
2. Créez une branche dédiée à votre modification.
3. Ouvrez une Pull Request en décrivant clairement le changement et son intérêt.

## Règles de sécurité pour les contributions

Ce projet enseigne la protection contre la surveillance. Le dépôt lui-même applique une discipline stricte :

- **Aucun secret dans le code.** Jamais de mot de passe, clé d'API, jeton ou identifiant en clair, même temporaire. Tout secret passe par des variables d'environnement non versionnées.
- **Aucune donnée personnelle.** Pas de nom réel, d'adresse, de numéro de téléphone ou d'email privé dans le code, les commits ou les exemples.
- **Aucun détail d'infrastructure.** Ne mentionnez pas d'hébergeur, d'adresse IP, de nom de serveur, de configuration réseau ni d'outil de déploiement.
- **Métadonnées.** Si vous joignez une image, retirez ses métadonnées (données EXIF) au préalable.

Une contribution qui introduit l'un de ces éléments ne sera pas fusionnée tant qu'il n'est pas retiré.

## Signaler un problème

Pour signaler une erreur ou proposer une idée sans toucher au code, ouvrez une Issue sur le dépôt. Pour un contact sensible, privilégiez les canaux chiffrés indiqués sur le site.
